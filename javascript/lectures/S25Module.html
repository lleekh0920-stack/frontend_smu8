제1장. JavaScript 모듈: Node.js와 브라우저 환경 구분

이 문서는 JavaScript 모듈을 수업할 때 Node.js 환경과 브라우저 환경을 구분하여 설명하기 위한 학습 자료이다. 공통 문법인 export와 import를 먼저 정리한 뒤, 각 실행 환경에서의 차이점을 비교한다.

1. 모듈의 기본 개념

JavaScript에서 모듈은 하나의 파일을 독립적인 코드 단위로 나누고, 필요한 값만 외부로 내보내거나 가져와 사용하는 구조를 말한다. 모듈을 사용하면 함수, 변수, 클래스 등을 파일별로 분리할 수 있으므로 코드 재사용성과 유지보수성이 좋아진다.

표준 JavaScript 모듈은 export와 import 문법을 사용한다. Node.js 공식 문서에서도 ECMAScript Module을 JavaScript 코드를 재사용하기 위한 공식 표준 형식으로 설명한다.

// math.js
export function add(a, b) {
  return a + b;
}
// app.js
import { add } from "./math.js";

console.log(add(3, 4));
핵심은 “파일을 나눈다”가 아니라, “파일 간에 무엇을 공개하고 무엇을 가져올지 명확히 정한다”는 점이다.

2. 공통 문법: export와 import

2.1. 이름 있는 내보내기

이름 있는 내보내기는 여러 개의 값을 각각 이름으로 내보낼 때 사용한다. 가져올 때는 중괄호 { } 안에 정확한 이름을 적어야 한다.

// user.js
export const name = "kim";

export function hello() {
  return "hello";
}
// app.js
import { name, hello } from "./user.js";

console.log(name);
console.log(hello());

2.2. 기본 내보내기

기본 내보내기는 한 파일에서 대표값 하나를 내보낼 때 사용한다. 가져올 때는 중괄호를 사용하지 않고, 가져오는 쪽에서 이름을 자유롭게 정할 수 있다.

// calculator.js
export default function multiply(a, b) {
  return a * b;
}
// app.js
import multiply from "./calculator.js";

console.log(multiply(2, 5));

2.3. 동적 import

정적 import는 파일의 최상단에서 모듈을 미리 가져오는 방식이고, 동적 import()는 필요한 시점에 비동기로 모듈을 가져오는 방식이다.

async function loadModule() {
  const module = await import("./math.js");
  console.log(module.add(1, 2));
}

loadModule();

3. Node.js에서의 모듈

Node.js에서는 크게 두 가지 모듈 방식을 구분해야 한다. 하나는 오래전부터 사용된 CommonJS 방식이고, 다른 하나는 표준 JavaScript 문법인 ECMAScript Module 방식이다.

3.1. CommonJS 방식

CommonJS는 Node.js에서 전통적으로 사용해 온 모듈 방식이다. 값을 내보낼 때는 module.exports 또는 exports를 사용하고, 가져올 때는 require()를 사용한다.

// math.cjs
function add(a, b) {
  return a + b;
}

module.exports = {
  add
};
// app.cjs
const { add } = require("./math.cjs");

console.log(add(3, 4));

CommonJS에서는 exports.add = ...처럼 짧게 쓸 수도 있다.

// math.cjs
exports.add = function (a, b) {
  return a + b;
};

다만 exports는 module.exports를 가리키는 편의 변수이므로, exports = { ... }처럼 새 객체를 직접 대입하면 의도한 대로 내보내지지 않는다.

// 잘못된 예
exports = {
  add(a, b) {
    return a + b;
  }
};
// 올바른 예
module.exports = {
  add(a, b) {
    return a + b;
  }
};

3.2. Node.js의 ES Module 방식

Node.js에서 ES Module을 사용하려면 파일 확장자나 package.json 설정을 통해 해당 파일이 모듈임을 알려야 한다. Node.js는 .mjs 파일, 또는 가장 가까운 package.json에 "type": "module"이 있는 .js 파일을 ES Module로 처리한다. 반대로 .cjs 파일이나 "type": "commonjs"가 설정된 .js 파일은 CommonJS로 처리된다.

{
  "type": "module"
}
// math.js
export function add(a, b) {
  return a + b;
}
// app.js
import { add } from "./math.js";

console.log(add(10, 20));

Node.js의 ES Module에서는 상대 경로로 모듈을 가져올 때 파일 확장자를 명시해야 한다. 예를 들어 ./math가 아니라 ./math.js처럼 작성해야 한다.

// 올바른 예
import { add } from "./math.js";
// Node.js ES Module에서는 피해야 할 예
import { add } from "./math";

Node.js의 ES Module에서는 require, exports, module.exports, __filename, __dirname 같은 CommonJS 전용 변수를 그대로 사용할 수 없다.

// Node.js ES Module
console.log(import.meta.url);

Node.js의 내장 모듈을 ES Module에서 가져올 때는 node: 접두사를 사용하는 형태가 권장 예시로 자주 등장한다.

import fs from "node:fs/promises";

const text = await fs.readFile("./hello.txt", "utf-8");
console.log(text);

4. 브라우저에서의 모듈

브라우저에서는 HTML에서 JavaScript 파일을 모듈로 실행하도록 지정해야 한다. 가장 기본적인 방법은 <script type="module">을 사용하는 것이다.

<!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="UTF-8">
  <title>Browser Module</title>
</head>
<body>
  <script type="module" src="./main.js"></script>
</body>
</html>
// math.js
export function add(a, b) {
  return a + b;
}
// main.js
import { add } from "./math.js";

console.log(add(5, 7));

브라우저에서 모듈을 사용할 때는 보통 로컬 파일을 직접 여는 file:// 방식이 아니라 개발 서버를 통해 실행해야 한다. 로컬 HTML 파일을 file:// URL로 열면 JavaScript 모듈의 보안 요구사항 때문에 CORS 오류가 발생할 수 있다. 또한 모듈은 자동으로 strict mode로 실행되고, 일반 스크립트와 달리 defer를 따로 붙이지 않아도 지연 실행되며, 여러 번 참조되어도 한 번만 실행된다.

브라우저의 모듈 경로는 URL 기준으로 해석된다. 따라서 상대 경로를 사용할 때는 ./math.js, ../utils/math.js처럼 명확한 경로를 작성하는 것이 좋다. 브라우저 환경에서는 Node.js처럼 "lodash" 같은 bare module specifier를 바로 해석할 수 없으며, 이를 사용하려면 import map 또는 번들러 설정이 필요하다.

<script type="importmap">
{
  "imports": {
    "utils": "./utils/index.js"
  }
}
</script>

<script type="module">
  import { hello } from "utils";

  console.log(hello());
</script>
