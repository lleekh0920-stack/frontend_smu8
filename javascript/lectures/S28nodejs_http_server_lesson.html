Chapter 01
Node.js란 무엇인가

1. Node.js의 개념

Node.js는 자바스크립트를 브라우저 밖에서 실행할 수 있게 해주는 실행 환경이다.

원래 자바스크립트는 브라우저 안에서 주로 실행되었다. 예를 들어 버튼 클릭 처리, HTML 제어, 화면 변경 같은 작업을 담당했다.

alert("안녕하세요");
document.querySelector("h1");
button.addEventListener("click", function () {});

하지만 Node.js를 사용하면 자바스크립트로 서버 프로그램도 만들 수 있다.

구분 역할
브라우저에서 사용하는 JavaScript 화면 처리, 버튼 클릭 처리, 입력값 검사, HTML/CSS 제어
Node.js에서 사용하는 JavaScript 서버 만들기, 파일 읽기, 데이터 처리, 네트워크 통신, API 만들기
핵심 정리

Node.js는 자바스크립트를 서버 개발에도 사용할 수 있게 해주는 실행 환경이다.

Chapter 02
서버란 무엇인가

1. 서버의 개념

서버는 요청을 받고 응답을 보내는 프로그램이다.

브라우저 → 요청 Request → 서버 브라우저 ← 응답 Response ← 서버

예를 들어 사용자가 브라우저 주소창에 다음 주소를 입력했다고 하자.

http://localhost:3000

그러면 브라우저는 서버에게 요청한다.

"3000번 포트에서 실행 중인 서버야, 나에게 응답을 보내줘."

서버는 브라우저에게 응답한다.

"Hello Node.js"

그러면 브라우저 화면에 응답 내용이 출력된다.

Chapter 03
Node.js 서버의 장점

1. 자바스크립트 하나로 개발 가능

Node.js를 사용하면 프론트엔드와 백엔드를 모두 자바스크립트로 만들 수 있다.

프론트엔드: JavaScript 백엔드: JavaScript

2. 비동기 처리에 강하다

Node.js는 비동기 방식에 강하다.

예를 들어 서버가 파일을 읽고 있는 동안에도 다른 요청을 처리할 수 있다.

파일 읽기 요청 처리 중 → 다른 사용자 요청도 처리 가능

그래서 Node.js는 웹 서버, API 서버, 채팅 서버, 파일 처리 서버, 실시간 데이터 서버에 잘 어울린다.

3. 짧은 코드로 서버를 만들 수 있다

Node.js에서는 기본 HTTP 서버를 짧은 코드로 만들 수 있다.

import http from "http";

const server = http.createServer((req, res) => {
    res.end("Hello Node.js");
});

server.listen(3000);

Chapter 06
localhost와 port 이해하기

1. localhost란?

localhost는 현재 내 컴퓨터를 의미한다.

localhost = 내 컴퓨터

즉, 다음 주소는 내 컴퓨터에서 실행 중인 서버에 접속한다는 뜻이다.

http://localhost:3000

2. port란?

포트는 서버 프로그램의 번호이다.

컴퓨터 안에서는 여러 서버 프로그램이 동시에 실행될 수 있다.

3000번 포트: Node.js 서버 3306번 포트: MySQL 5500번 포트: Live Server

그래서 3000은 Node.js 서버가 실행 중인 번호라고 볼 수 있다.

Chapter 07
요청 Request와 응답 Response

1. 요청 객체 req

req는 브라우저가 서버에게 보낸 요청 정보를 담고 있다.

const server = http.createServer((req, res) => {
    console.log(req.method);
    console.log(req.url);

    res.end("ok");
});

브라우저에서 다음 주소로 접속해보자.

http://localhost:3000/user

터미널에는 다음과 비슷한 내용이 출력된다.

GET
/user

2. req.method

req.method는 요청 방식을 의미한다.

요청 방식 의미
GET 데이터 조회
POST 데이터 등록
PUT 데이터 수정
DELETE 데이터 삭제

3. req.url

req.url은 사용자가 요청한 주소 경로를 의미한다.

다음 주소로 요청하면

http://localhost:3000/user

req.url 값은 다음과 같다.

/user

다음 주소로 요청하면

http://localhost:3000/user?name=aaa

req.url 값은 다음과 같다.

/user?name=aaa

4. 응답 객체 res

res는 서버가 브라우저에게 응답을 보낼 때 사용한다.

가장 기본적인 응답은 res.end()이다.

res.end("응답 완료");

Chapter 09
라우팅 처리하기

1. 라우팅이란?

라우팅이란 주소에 따라 다른 응답을 보내는 것이다.

/ → 메인 페이지 /user → 유저 페이지 /about → 소개 페이지

2. URL별 응답하기

server.js 파일을 다음처럼 작성한다.

import http from "http";

const server = http.createServer((req, res) => {
    const url = new URL(req.url, "http://localhost:3000");

    if (url.pathname === "/") {
        res.end("메인 페이지");
    } else if (url.pathname === "/user") {
        res.end("유저 페이지");
    } else if (url.pathname === "/about") {
        res.end("소개 페이지");
    } else {
        res.statusCode = 404;
        res.end("페이지 없음");
    }
});

server.listen(3000, () => {
    console.log("서버 실행 중: http://localhost:3000");
});

Chapter 10
GET Query String 처리하기

1. Query String이란?

Query String은 URL 뒤에 데이터를 붙여서 서버로 보내는 방식이다.

http://localhost:3000/user?name=aaa&age=20

구조는 다음과 같다.

주소?키=값&키=값

위 주소에서는 다음 데이터를 서버로 보낸다.

name = aaa
age = 20

2. searchParams 사용하기

Query String 값은 url.searchParams.get()으로 읽을 수 있다.

const name = url.searchParams.get("name");
const age = url.searchParams.get("age");

3. Query String 실습 코드

import http from "http";

const server = http.createServer((req, res) => {
    const url = new URL(req.url, "http://localhost:3000");

    if (url.pathname === "/user") {
        const name = url.searchParams.get("name");
        const age = url.searchParams.get("age");

        console.log("name:", name);
        console.log("age:", age);

        res.end(`name은 ${name}, age는 ${age}`);
    } else {
        res.end("메인 페이지");
    }
});

server.listen(3000, () => {
    console.log("서버 실행 중: http://localhost:3000");
});

브라우저에서 다음 주소로 접속한다.

http://localhost:3000/user?name=aaa&age=20

브라우저 결과:

name은 aaa, age는 20

터미널 결과:

name: aaa
age: 20

Chapter 11
HTML 응답하기

1. 문자열로 HTML 응답하기

서버는 단순한 텍스트뿐만 아니라 HTML도 응답할 수 있다.

HTML을 응답할 때는 Content-Type을 설정하는 것이 좋다.

res.setHeader("Content-Type", "text/html; charset=utf-8");

charset=utf-8을 설정하면 한글이 깨지지 않는다.

2. HTML 응답 코드

import http from "http";

const server = http.createServer((req, res) => {
    res.setHeader("Content-Type", "text/html; charset=utf-8");

    res.end(`
        <!DOCTYPE html>
        <html lang="ko">
        <head>
            <meta charset="UTF-8">
            <title>Node 서버</title>
        </head>
        <body>
            <h1>안녕하세요</h1>
            <p>Node.js 서버에서 보낸 HTML입니다.</p>
        </body>
        </html>
    `);
});

server.listen(3000, () => {
    console.log("서버 실행 중: http://localhost:3000");
});

브라우저 화면에는 HTML이 적용된 결과가 출력된다.

Chapter 13
JSON 응답하기

1. JSON이란?

JSON은 데이터를 문자열 형태로 표현하는 형식이다.

자바스크립트 객체와 비슷하게 생겼지만, 실제로는 문자열 데이터이다.

자바스크립트 객체:

const user = {
    name: "aaa",
    age: 20
};

JSON 문자열:

{
  "name": "aaa",
  "age": 20
}

2. JSON.stringify()

서버에서 객체를 바로 응답할 수는 없다.

객체를 JSON 문자열로 바꿔서 보내야 한다.

JSON.stringify(user)

3. JSON 객체 응답하기

import http from "http";

const server = http.createServer((req, res) => {

        name: "aaa",

server.listen(3000, () => {
    console.log("서버 실행 중: http://localhost:3000");

Chapter 14
최종 실습 코드

아래 코드는 지금까지 배운 내용을 하나로 합친 예제이다.

server.js

import http from "http";
import fs from "fs/promises";

const PORT = 3000;

const server = http.createServer(async (req, res) => {
    const url = new URL(req.url, `http://localhost:${PORT}`);

    console.log("요청 방식:", req.method);
    console.log("요청 경로:", url.pathname);

    if (url.pathname === "/") {
        res.setHeader("Content-Type", "text/plain; charset=utf-8");
        res.end("메인 페이지");
    }

    else if (url.pathname === "/user") {
        const name = url.searchParams.get("name");
        const age = url.searchParams.get("age");

        res.setHeader("Content-Type", "text/plain; charset=utf-8");
        res.end(`유저 페이지 / name: ${name}, age: ${age}`);
    }

    else if (url.pathname === "/html") {
        res.setHeader("Content-Type", "text/html; charset=utf-8");
        res.end(`
            <!DOCTYPE html>
            <html lang="ko">
            <head>
                <meta charset="UTF-8">
                <title>HTML 응답</title>
            </head>
            <body>
                <h1>Node.js HTML 응답</h1>
                <p>서버에서 HTML 문자열을 보냈습니다.</p>
            </body>
            </html>
        `);
    }

    else if (url.pathname === "/file") {
        try {
            const html = await fs.readFile("./index.html", "utf-8");

            res.setHeader("Content-Type", "text/html; charset=utf-8");
            res.end(html);
        } catch (error) {
            res.statusCode = 500;
            res.setHeader("Content-Type", "text/plain; charset=utf-8");
            res.end("HTML 파일을 읽을 수 없습니다.");
        }
    }

    else if (url.pathname === "/api/user") {
        const user = {
            name: "aaa",
            age: 20
        };

        res.setHeader("Content-Type", "application/json; charset=utf-8");
        res.end(JSON.stringify(user));
    }

    else if (url.pathname === "/api/users") {
        const users = [
            { name: "aaa", age: 10 },
            { name: "bbb", age: 20 },
            { name: "ccc", age: 30 }
        ];

        res.setHeader("Content-Type", "application/json; charset=utf-8");
        res.end(JSON.stringify(users));
    }

    else {
        res.statusCode = 404;
        res.setHeader("Content-Type", "text/plain; charset=utf-8");
        res.end("페이지를 찾을 수 없습니다.");
    }
});

server.listen(PORT, () => {
    console.log(`서버 실행 중: http://localhost:${PORT}`);
});

문제 2 정답

import http from "http";

const server = http.createServer((req, res) => {
    const url = new URL(req.url, "http://localhost:3000");

        res.end("유저 페이지");

문제 6 정답

import http from "http";

const server = http.createServer((req, res) => {
    const url = new URL(req.url, "http://localhost:3000");

    res.setHeader("Content-Type", "text/plain; charset=utf-8");

    if (url.pathname === "/") {
        res.end("메인 페이지");
    } else if (url.pathname === "/user") {
        res.end("유저 페이지");
    } else if (url.pathname === "/product") {
        res.end("상품 페이지");
    } else {
        res.statusCode = 404;
        res.end("페이지 없음");
    }
});

server.listen(3000);

수업 마무리 정리

  • Node.js는 브라우저 밖에서 자바스크립트를 실행하는 환경이다.
  • 서버는 요청을 받고 응답을 보내는 프로그램이다.
  • http 모듈을 사용하면 Node.js 기본 서버를 만들 수 있다.
  • req는 요청 정보를 담고 있다.
  • res는 응답을 보낼 때 사용한다.
  • URL 객체를 사용하면 요청 주소를 쉽게 분석할 수 있다.
  • url.pathname으로 경로를 확인한다.
  • url.searchParams.get()으로 Query String 값을 읽는다.
  • res.setHeader()로 응답 형식을 설정한다.
  • JSON 응답은 JSON.stringify()를 사용한다.
