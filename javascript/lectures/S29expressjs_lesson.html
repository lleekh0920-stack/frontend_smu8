Express.js 서버 수업 교안

Node.js 기본 HTTP 서버 다음 단계로, Express.js를 사용하여 라우팅, Query String, URL Parameter, HTML 응답, JSON 응답, POST 요청, 미들웨어를 학습한다.

수업 목표

  1. Express.js가 무엇인지 설명할 수 있다.
  2. Node.js 기본 HTTP 서버와 Express 서버의 차이를 이해할 수 있다.
  3. Express 프로젝트를 생성하고 서버를 실행할 수 있다.
  4. app.get(), app.post()로 라우팅을 처리할 수 있다.
  5. req.query, req.params, req.body를 사용할 수 있다.
  6. res.send(), res.json(), res.status()로 응답할 수 있다.
  7. HTML 문자열 응답과 HTML 파일 응답을 구현할 수 있다.
  8. 간단한 REST API 서버를 만들 수 있다.

Chapter 01
Express.js란 무엇인가

1. Express.js의 개념

Express.js는 Node.js에서 서버를 더 쉽고 간단하게 만들 수 있게 해주는 웹 프레임워크이다.

이전 수업에서는 Node.js 기본 http 모듈로 서버를 만들었다.

import http from "http";

const server = http.createServer((req, res) => {
    res.end("Hello Node.js");
});

server.listen(3000);

Node.js 기본 서버는 URL 처리, JSON 처리, POST 데이터 처리, 라우팅 처리를 직접 작성해야 한다.

Express를 사용하면 서버 코드를 훨씬 간단하게 만들 수 있다.

import express from "express";

const app = express();

app.get("/", (req, res) => {
    res.send("Hello Express");
});

app.listen(3000);
핵심 정리

Express.js는 Node.js 서버 개발을 쉽게 해주는 도구이다.

Chapter 02
Node.js 기본 서버와 Express 비교

1. Node.js 기본 HTTP 서버

Node.js 기본 서버에서는 URL을 직접 분석해야 한다.

import http from "http";

const server = http.createServer((req, res) => {
    const url = new URL(req.url, "http://localhost:3000");

    if (url.pathname === "/") {
        res.end("메인 페이지");
    } else if (url.pathname === "/user") {
        res.end("유저 페이지");
    } else {
        res.statusCode = 404;
        res.end("페이지 없음");
    }
});

server.listen(3000);

2. Express 서버

Express에서는 라우팅을 더 간단하게 작성할 수 있다.

import express from "express";

const app = express();

app.get("/", (req, res) => {
    res.send("메인 페이지");
});

app.get("/user", (req, res) => {
    res.send("유저 페이지");
});

app.listen(3000);
구분 Node.js 기본 HTTP 서버 Express 서버
서버 생성 http.createServer() express()
라우팅 직접 조건문 작성 app.get(), app.post()
Query String new URL() 사용 req.query 사용
JSON 응답 JSON.stringify() res.json()
Node 기본 서버: 직접 URL 분석 직접 조건문 작성 직접 응답 처리 Express 서버: app.get("/주소", 함수) app.post("/주소", 함수) res.send() res.json()

Chapter 06
Query String 처리하기

1. Query String 복습

Query String은 URL 뒤에 데이터를 붙여서 보내는 방식이다.

http://localhost:3000/user?name=aaa&age=20

데이터 구조:

name = aaa
age = 20

2. Node 기본 서버에서 Query String 읽기

이전 수업에서는 이렇게 작성했다.

const url = new URL(req.url, "http://localhost:3000");

const name = url.searchParams.get("name");

3. Express에서 Query String 읽기

Express에서는 req.query를 사용한다.

app.get("/user", (req, res) => {
    const name = req.query.name;
    const age = req.query.age;

    res.send(`name: ${name}, age: ${age}`);
});

4. 전체 코드

import express from "express";

const app = express();
const PORT = 3000;

app.get("/user", (req, res) => {
    const name = req.query.name;
    const age = req.query.age;

    console.log("name:", name);
    console.log("age:", age);

    res.send(`name은 ${name}, age는 ${age}`);
});

app.listen(PORT, () => {
    console.log(`서버 실행 중: http://localhost:${PORT}`);
});

5. 테스트

http://localhost:3000/user?name=aaa&age=20

브라우저 결과:

name은 aaa, age는 20

터미널 결과:

name: aaa
age: 20

Chapter 07
URL Parameter 처리하기

1. URL Parameter란?

URL Parameter는 주소 자체에 값을 포함하는 방식이다.

http://localhost:3000/user/1
http://localhost:3000/user/2
http://localhost:3000/user/3

여기서 1, 2, 3이 데이터가 된다.

2. req.params 사용하기

Express에서는 URL Parameter를 req.params로 읽는다.

app.get("/user/:id", (req, res) => {
    const id = req.params.id;

    res.send(`유저 ID: ${id}`);
});

3. 전체 코드

import express from "express";

const app = express();
const PORT = 3000;

app.get("/user/:id", (req, res) => {
    const id = req.params.id;

    res.send(`유저 ID는 ${id}입니다.`);
});

app.listen(PORT, () => {
    console.log(`서버 실행 중: http://localhost:${PORT}`);
});

4. 테스트

결과:

Chapter 08
HTML 응답하기

1. res.send()로 HTML 문자열 응답하기

Express에서는 res.send()로 문자열이나 HTML을 응답할 수 있다.

app.get("/html", (req, res) => {
    res.send("<h1>안녕하세요</h1>");
});

2. HTML 문자열 응답 전체 코드

import express from "express";

const app = express();
const PORT = 3000;

app.get("/html", (req, res) => {
    res.send(`
        <!DOCTYPE html>
        <html lang="ko">
        <head>
            <meta charset="UTF-8">
            <title>Express HTML</title>
        </head>
        <body>
            <h1>Express HTML 응답</h1>
            <p>Express 서버에서 보낸 HTML 문자열입니다.</p>
        </body>
        </html>
    `);
});

app.listen(PORT, () => {
    console.log(`서버 실행 중: http://localhost:${PORT}`);
});
추가 설명

HTML 문자열을 직접 응답하는 방식은 간단한 예제에는 좋다. 하지만 실제 화면이 길어지면 문자열 안에 HTML을 모두 작성하기 불편하다. 이럴 때는 HTML 파일을 따로 만들고, 서버가 그 파일을 응답하게 만들 수 있다.

3. HTML 파일 응답하기

이번에는 index.html 파일을 만들고, Express 서버가 그 파일을 브라우저에 보내도록 한다.

프로젝트 구조

express-server-study
├── package.json
├── app.js
└── index.html

index.html

<!DOCTYPE html>
<html lang="ko">
<head>
    <meta charset="UTF-8">
    <title>Express HTML 파일</title>
</head>
<body>
    <h1>HTML 파일 응답</h1>
    <p>이 문서는 index.html 파일에서 읽어온 내용입니다.</p>
</body>
</html>

app.js

res.sendFile()을 사용하면 HTML 파일을 응답할 수 있다.

import express from "express";

const app = express();
const PORT = 3000;

app.get("/file", (req, res) => {
    res.sendFile("index.html", {
        root: process.cwd()
    });
});

app.listen(PORT, () => {
    console.log(`서버 실행 중: http://localhost:${PORT}`);
});
코드 설명
  • res.sendFile(): 파일을 브라우저로 응답한다.
  • "index.html": 응답할 파일 이름이다.
  • root: process.cwd(): 현재 프로젝트 폴더를 기준으로 파일을 찾겠다는 의미이다.

테스트

브라우저 결과:

HTML 파일 응답
이 문서는 index.html 파일에서 읽어온 내용입니다.
주의

터미널에서 서버를 실행할 때는 index.html 파일이 있는 프로젝트 폴더에서 node app.js를 실행해야 한다.

Chapter 10
POST 요청 처리하기

1. GET과 POST 차이

방식 특징 주 사용 목적
GET 주소에 데이터를 붙여서 보냄 조회
POST 요청 body에 데이터를 담아서 보냄 등록
GET /users 사용자 목록 조회 POST /users 새 사용자 등록

2. express.json() 사용하기

POST 요청으로 JSON 데이터를 받을 때는 다음 코드가 필요하다.

app.use(express.json());

express.json()은 요청 body에 들어온 JSON 데이터를 읽어서 req.body에 넣어준다.

3. POST 요청 코드

import express from "express";

const app = express();
const PORT = 3000;

app.use(express.json());

app.post("/users", (req, res) => {
    console.log(req.body);

    const name = req.body.name;
    const age = req.body.age;

    res.json({
        message: "사용자 등록 완료",
        user: {
            name: name,
            age: age
        }
    });
});

app.listen(PORT, () => {
    console.log(`서버 실행 중: http://localhost:${PORT}`);
});

4. 요청 JSON 예시

Chapter 11
상태 코드 응답하기

1. 상태 코드란?

상태 코드는 서버 응답의 상태를 숫자로 표현한 것이다.

상태 코드 의미
200 정상
201 생성 완료
400 잘못된 요청
404 찾을 수 없음
500 서버 오류

2. res.status()

Express에서는 res.status()로 상태 코드를 설정한다.

res.status(404).send("페이지 없음");

JSON 응답과 함께 사용할 수도 있다.

res.status(201).json({
    message: "등록 완료"
});

3. 404 응답 처리

app.get("/user", (req, res) => {
    res.send("유저 페이지");
});

app.use((req, res) => {
    res.status(404).send("페이지를 찾을 수 없습니다.");
});

404 처리는 보통 모든 라우트 아래쪽에 작성한다.

Chapter 12
미들웨어 Middleware

1. 미들웨어란?

미들웨어는 요청과 응답 사이에서 실행되는 함수이다.

요청 ↓ 미들웨어 ↓ 라우터 ↓ 응답

미들웨어에서는 다음과 같은 작업을 할 수 있다.

  • 요청 로그 출력
  • JSON body 처리
  • 로그인 검사
  • 에러 처리
  • 정적 파일 제공

2. app.use()

미들웨어는 보통 app.use()로 등록한다.

app.use((req, res, next) => {
    console.log("요청 들어옴");
    next();
});

next()를 호출해야 다음 미들웨어나 라우터로 넘어간다.

3. 요청 로그 미들웨어

import express from "express";

const app = express();
const PORT = 3000;

app.use((req, res, next) => {
    console.log("요청 방식:", req.method);
    console.log("요청 주소:", req.url);
    next();
});

app.get("/", (req, res) => {
    res.send("메인 페이지");
});

app.get("/user", (req, res) => {
    res.send("유저 페이지");
});

app.listen(PORT, () => {
    console.log(`서버 실행 중: http://localhost:${PORT}`);
});

Chapter 13
에러 처리 미들웨어

1. 에러 처리 미들웨어 구조

Express의 에러 처리 미들웨어는 인자가 4개이다.

app.use((err, req, res, next) => {
    res.status(500).send("서버 오류");
});

2. 에러 처리 예제

import express from "express";

const app = express();
const PORT = 3000;

app.get("/", (req, res) => {
    res.send("메인 페이지");
});

app.get("/error", (req, res) => {
    throw new Error("강제 에러 발생");
});

app.use((err, req, res, next) => {
    console.error(err.message);

    res.status(500).json({
        message: "서버 내부 오류가 발생했습니다."
    });
});

app.listen(PORT, () => {
    console.log(`서버 실행 중: http://localhost:${PORT}`);
});

테스트 주소:

http://localhost:3000/error

결과:

{
  "message": "서버 내부 오류가 발생했습니다."
}

    { id: 1, name: "aaa", age: 10 },
    { id: 2, name: "bbb", age: 20 },
    { id: 3, name: "ccc", age: 30 }
];

app.get("/", (req, res) => {
    res.send("Express 사용자 API 서버");
});

app.get("/html", (req, res) => {
    res.send(`
        <!DOCTYPE html>
        <html lang="ko">
        <head>
            <meta charset="UTF-8">
            <title>HTML 문자열 응답</title>
        </head>
        <body>
            <h1>Express HTML 문자열 응답</h1>
            <p>서버에서 HTML 문자열을 직접 보냈습니다.</p>
        </body>
        </html>
    `);
});

app.get("/file", (req, res) => {
    res.sendFile("index.html", {
        root: process.cwd()
    });
});

app.get("/users", (req, res) => {
    res.json(users);
});

app.get("/users/:id", (req, res) => {
    const id = Number(req.params.id);

    const user = users.find((item) => item.id === id);

    if (!user) {
        return res.status(404).json({
            message: "사용자를 찾을 수 없습니다."
        });
    }

    res.json(user);
});

app.post("/users", (req, res) => {
    const name = req.body.name;
    const age = req.body.age;

    if (!name || !age) {
        return res.status(400).json({
            message: "name과 age를 입력해야 합니다."
        });
    }

    const newUser = {
        id: users.length + 1,
        name: name,
        age: age
    };

    users.push(newUser);

    res.status(201).json({
        message: "사용자 등록 완료",
        user: newUser
    });
});

app.use((req, res) => {
    res.status(404).json({
        message: "페이지를 찾을 수 없습니다."
    });
});

app.listen(PORT, () => {
    console.log(`서버 실행 중: http://localhost:${PORT}`);
});

수업 마무리 정리

  • Express.js는 Node.js 서버 개발을 쉽게 해주는 프레임워크이다.
  • express()로 Express 앱을 만든다.
  • app.listen()으로 서버를 실행한다.
  • app.get()은 GET 요청을 처리한다.
  • app.post()는 POST 요청을 처리한다.
  • req.query는 Query String 값을 읽을 때 사용한다.
  • req.params는 URL Parameter 값을 읽을 때 사용한다.
  • req.body는 POST 요청 body 값을 읽을 때 사용한다.
  • POST JSON 데이터를 읽으려면 app.use(express.json())이 필요하다.
  • res.send()는 문자열이나 HTML을 응답할 때 사용한다.
  • res.sendFile()은 HTML 파일 같은 실제 파일을 응답할 때 사용한다.
  • res.json()은 JSON 데이터를 응답할 때 사용한다.
  • res.status()는 상태 코드를 설정할 때 사용한다.
  • app.use()는 미들웨어를 등록할 때 사용한다.
