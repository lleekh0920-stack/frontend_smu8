Chapter 01
React란 무엇인가

1. React의 개념

React는 사용자 인터페이스 UI를 만들기 위한 JavaScript 라이브러리이다.

React의 핵심은 화면을 여러 개의 컴포넌트 Component로 나누어 관리하는 것이다.

전체 페이지 ├── Header 컴포넌트 ├── Menu 컴포넌트 ├── Content 컴포넌트 ├── Button 컴포넌트 └── Footer 컴포넌트

React에서는 이런 조각들을 함수로 만들 수 있다.

function Header() {
    return <h1>React 수업</h1>;
}

function Button() {
    return <button>클릭</button>;
}

2. React는 프레임워크인가, 라이브러리인가?

React는 보통 라이브러리라고 부른다. React 자체가 전체 웹 애플리케이션의 모든 구조를 강제하지 않고, 주로 UI를 만드는 역할을 담당하기 때문이다.

React가 주로 담당하는 것: 화면 UI 만들기 React가 직접 강제하지 않는 것: 라우팅 서버 통신 방식 프로젝트 구조 상태 관리 방식 빌드 방식

그래서 React는 필요에 따라 다른 도구와 함께 사용한다.

React + Vite → 웹 프로젝트 React + Next.js → 풀스택 / SSR 웹 프로젝트 React + Expo → 모바일 앱 / 웹 앱 프로젝트 React + React Router → SPA 라우팅

Chapter 02
React 개발사와 나오게 된 배경

1. React의 출발

React는 처음에 Facebook, 현재 Meta 내부에서 만들어졌다. 복잡한 화면을 더 예측 가능하게 만들고, 데이터 변화에 따라 UI를 효율적으로 업데이트하기 위해 만들어졌다.

현재 거버넌스 참고

2026년 2월 24일 React Foundation이 Linux Foundation 산하에 공식 출범했고, Meta가 React 프로젝트를 Linux Foundation에 기여했다. 수업에서는 “처음 개발은 Facebook/Meta, 현재는 React Foundation 중심의 오픈 거버넌스 체계로 이동”이라고 설명하면 좋다.

2. React가 나오게 된 배경

React가 나오기 전에는 화면을 바꿀 때 개발자가 직접 DOM을 조작하는 경우가 많았다.

const title = document.querySelector("#title");
title.textContent = "변경된 제목";

작은 화면에서는 괜찮지만, Facebook처럼 화면이 복잡하고 데이터가 자주 바뀌는 서비스에서는 문제가 생긴다.

데이터가 자주 바뀜 화면도 자주 바뀜 DOM을 직접 조작해야 함 코드가 복잡해짐 버그가 많아짐

React는 이런 문제를 줄이기 위해 등장했다.

React의 핵심 생각

개발자는 현재 상태에 맞는 화면을 선언한다. 실제 DOM을 어떻게 바꿀지는 React가 처리한다.

Chapter 03
컴포넌트와 선언형 UI

1. 컴포넌트 기반

React는 화면을 컴포넌트 단위로 나눈다.

function App() {
    return (
        <div>
            <Header />
            <Content />
            <Footer />
        </div>
    );
}

컴포넌트 기반으로 만들면 코드를 나누기 쉽고, 재사용하기 쉽고, 화면 구조를 이해하기 쉽다.

2. 선언형 UI

React는 선언형 방식으로 화면을 만든다.

명령형 방식: 1. 버튼을 찾는다. 2. 글자를 찾는다. 3. 값을 바꾼다. 4. 색상을 바꾼다. 선언형 방식: 현재 count 값이 0이면 화면에 0을 보여준다. count 값이 1이면 화면에 1을 보여준다.

React에서는 상태가 바뀌면 화면이 다시 렌더링된다.

function Counter() {
    const [count, setCount] = React.useState(0);

    return (
        <div>
            <h1>{count}</h1>
            <button onClick={() => setCount(count + 1)}>
                증가
            </button>
        </div>
    );
}

Chapter 04
JSX

1. JSX란?

JSX는 JavaScript 코드 안에서 HTML과 비슷한 마크업을 작성할 수 있게 해주는 문법 확장이다.

const element = <h1>안녕하세요</h1>;

위 코드는 문자열도 아니고 일반 HTML도 아니다. JavaScript 안에 작성된 JSX 문법이다.

쉽게 말하면

JSX는 “JavaScript 안에서 HTML처럼 화면 구조를 작성하는 문법”이다.

2. JSX를 사용하는 이유

React 컴포넌트는 화면 구조와 JavaScript 로직이 함께 움직이는 경우가 많다.

JSX를 사용하면 컴포넌트 안에서 화면 구조를 직관적으로 작성할 수 있다.

function Profile() {
    const name = "홍길동";

    return (
        <div>
            <h1>{name}</h1>
            <p>React 수업을 듣고 있습니다.</p>
        </div>
    );
}

3. JSX 기본 규칙 1 — 하나의 부모 요소로 감싸기

컴포넌트가 여러 요소를 반환할 때는 하나의 부모 요소로 감싸야 한다.

잘못된 예:

function App() {
    return (
        <h1>제목</h1>
        <p>내용</p>
    );
}

올바른 예:

function App() {
    return (
        <div>
            <h1>제목</h1>
            <p>내용</p>
        </div>
    );
}

또는 Fragment를 사용할 수 있다.

function App() {
    return (
        <>
            <h1>제목</h1>
            <p>내용</p>
        </>
    );
}

4. JSX 기본 규칙 2 — JavaScript 값은 중괄호 사용

JSX 안에서 JavaScript 값을 출력할 때는 { }를 사용한다.

function App() {
    const name = "홍길동";
    const age = 20;

    return (
        <div>
            <h1>{name}</h1>
            <p>{age}살입니다.</p>
            <p>내년에는 {age + 1}살입니다.</p>
        </div>
    );
}

5. JSX 기본 규칙 3 — class 대신 className

HTML에서는 class를 사용하지만, JSX에서는 className을 사용한다.

function App() {
    return (
        <div className="box">
            <h1 className="title">React</h1>
        </div>
    );
}

6. JSX 기본 규칙 4 — 이벤트는 camelCase

HTML에서는 onclick처럼 작성하지만, JSX에서는 onClick처럼 camelCase로 작성한다.

function App() {
    function handleClick() {
        alert("클릭했습니다.");
    }

    return (
        <button onClick={handleClick}>
            클릭
        </button>
    );
}

7. JSX 기본 규칙 5 — 스타일은 객체로 작성

JSX에서 인라인 스타일을 작성할 때는 문자열이 아니라 객체를 사용한다.

function App() {
    return (
        <h1 style={{ color: "blue", fontSize: "30px" }}>
            React 스타일
        </h1>
    );
}

8. JSX에서 조건부 렌더링

조건에 따라 다른 화면을 보여줄 수 있다.

function LoginMessage() {
    const isLogin = true;

    return (
        <div>
            {isLogin ? (
                <p>로그인 상태입니다.</p>
            ) : (
                <p>로그인이 필요합니다.</p>
            )}
        </div>
    );
}

9. JSX에서 배열 출력하기

배열은 map()을 사용해 화면에 출력한다. 반복 출력할 때는 각 항목에 key를 넣는 것이 중요하다.

function UserList() {
    const users = [
        { id: 1, name: "aaa" },
        { id: 2, name: "bbb" },
        { id: 3, name: "ccc" }
    ];

    return (
        <div>
            <h1>사용자 목록</h1>

            {users.map((user) => (
                <p key={user.id}>{user.name}</p>
            ))}
        </div>
    );
}
JSX 핵심 정리
  • JSX는 JavaScript 안에서 HTML처럼 화면을 작성하는 문법이다.
  • 여러 요소를 반환할 때는 하나의 부모 요소로 감싼다.
  • JavaScript 값은 { } 안에 작성한다.
  • class 대신 className을 사용한다.
  • 이벤트는 onClick, onChange처럼 camelCase로 작성한다.
  • 배열 출력은 map()을 사용한다.

Chapter 05
Virtual DOM, 가상 DOM

1. DOM이란?

DOM은 HTML 문서를 JavaScript가 다룰 수 있도록 객체 구조로 표현한 것이다.

<h1>안녕하세요</h1>
<p>React 수업입니다.</p>

브라우저는 위 HTML을 다음과 같은 구조로 이해한다.

document └── body ├── h1 └── p

JavaScript는 DOM을 이용해 화면을 변경할 수 있다.

document.querySelector("h1").textContent = "변경됨";

2. Virtual DOM이란?

Virtual DOM은 실제 DOM을 바로 바꾸기 전에, React가 메모리 안에서 관리하는 가상의 UI 표현이라고 이해하면 된다.

1. 상태 state가 변경된다. 2. React가 새로운 화면 구조를 계산한다. 3. 이전 화면 구조와 새 화면 구조를 비교한다. 4. 실제로 바뀐 부분만 실제 DOM에 반영한다.

3. Virtual DOM 흐름

state 변경 ↓ 컴포넌트 다시 실행 ↓ 새로운 Virtual DOM 생성 ↓ 이전 Virtual DOM과 비교 ↓ 바뀐 부분 계산 ↓ 실제 DOM 업데이트
학생용 쉬운 설명

실제 칠판을 바로 지우고 다시 쓰는 것이 아니라, 먼저 연습장에 바뀔 내용을 계산한 뒤, 칠판에서는 진짜 바뀐 부분만 고치는 방식이다.

Chapter 06
SPA와 MPA

1. MPA란?

MPA는 Multi Page Application의 약자이다. 여러 개의 HTML 페이지로 구성된 전통적인 웹 방식이다.

/ ├── index.html ├── login.html ├── board.html └── detail.html

사용자가 페이지를 이동할 때마다 서버에서 새로운 HTML 문서를 받아온다.

/login 클릭 ↓ 서버에 login.html 요청 ↓ 브라우저가 새 페이지 로드

2. SPA란?

SPA는 Single Page Application의 약자이다. 처음에 하나의 HTML 문서를 받고, 이후 화면 변경은 JavaScript가 처리하는 방식이다.

index.html 하나 ↓ React JavaScript 실행 ↓ 화면을 컴포넌트로 바꿔가며 표시

예를 들어 사용자가 /login으로 이동해도 실제로 새로운 login.html을 받는 것이 아니라, React가 현재 화면을 로그인 컴포넌트로 바꿔준다.

/login 이동 ↓ 새 HTML 요청 X ↓ React가 Login 컴포넌트 표시

3. SPA와 MPA 비교

구분 MPA SPA
뜻 Multi Page Application Single Page Application
HTML 구조 여러 HTML 페이지 하나의 HTML 중심
페이지 이동 서버에서 새 HTML 로드 JavaScript로 화면 전환
새로고침 자주 발생 적게 발생
사용자 경험 전통적인 웹 느낌 앱처럼 부드러운 느낌
대표 예 회사 소개 사이트, 전통 게시판 대시보드, 관리자 페이지, 채팅 앱

Chapter 07
React를 라이브러리처럼 사용하는 법

1. 언제 이 방식을 쓰는가?

React는 꼭 전체 프로젝트로 시작하지 않아도 된다. 기존 HTML 페이지의 일부 영역에만 React를 붙일 수 있다.

기존 HTML 페이지 ├── 일반 HTML 영역 ├── 일반 CSS 영역 └── React 버튼 영역만 추가

이 방식은 다음 상황에 좋다.

  • React를 처음 실습할 때
  • 기존 HTML 페이지 일부에만 React를 붙이고 싶을 때
  • 빌드 도구 없이 React 개념을 보여주고 싶을 때

2. CDN으로 React 추가하기

아래 예제는 교육용으로 가장 단순하게 만든 방식이다.

파일 이름:

index.html

코드:

<!DOCTYPE html>
<html lang="ko">
<head>
    <meta charset="UTF-8">
    <title>React 라이브러리 방식</title>
</head>
<body>
    <h1>기존 HTML 페이지</h1>

    <div id="root"></div>

    <!-- React 라이브러리 추가 -->
    <script src="https://unpkg.com/react@18/umd/react.development.js"></script>
    <script src="https://unpkg.com/react-dom@18/umd/react-dom.development.js"></script>

    <!-- JSX를 브라우저에서 사용하기 위한 Babel -->
    <script src="https://unpkg.com/@babel/standalone/babel.min.js"></script>

    <script type="text/babel">
        function App() {
            const [count, setCount] = React.useState(0);

            return (
                <div>
                    <h2>React 영역</h2>
                    <p>count: {count}</p>
                    <button onClick={() => setCount(count + 1)}>
                        증가
                    </button>
                </div>
            );
        }

        const root = ReactDOM.createRoot(
            document.getElementById("root")
        );

        root.render(<App />);
    </script>
</body>
</html>

3. 코드 설명

<div id="root"></div>

React 컴포넌트가 들어갈 위치이다.

function App() {
    return <h2>React 영역</h2>;
}

React 컴포넌트이다.

const root = ReactDOM.createRoot(
    document.getElementById("root")
);

id="root"인 HTML 요소 안에 React를 실행할 준비를 한다.

root.render(<App />);

App 컴포넌트를 화면에 출력한다.

4. 라이브러리 방식의 장단점

장점 단점
설치가 간단하다.
기존 HTML에 React를 조금만 붙일 수 있다.
초보자 실습에 좋다. 		실무 프로젝트에는 부족하다.
파일 분리가 어렵다.
빌드 최적화가 어렵다.
라우팅, 상태 관리, 배포 구조를 갖추기 어렵다.
수업용 주의

CDN + Babel 방식은 React 개념을 빠르게 보여주기 위한 교육용 방식이다. 실제 프로젝트에서는 Vite, Next.js, Expo 같은 프로젝트 도구를 사용하는 것이 일반적이다.

Chapter 08
웹에서 Babel이란?

1. Babel의 개념

Babel은 JavaScript 컴파일러이다. 쉽게 말하면, 브라우저나 실행 환경이 바로 이해하기 어려운 최신 JavaScript 문법 또는 JSX 문법을 실행 가능한 JavaScript 코드로 바꿔주는 도구이다.

JSX / 최신 JavaScript 코드 ↓ Babel 변환 ↓ 브라우저가 실행할 수 있는 JavaScript 코드

2. 브라우저는 JSX를 그대로 이해하지 못한다

다음 코드는 React 개발자에게는 익숙하지만, 브라우저 입장에서는 일반 JavaScript 문법이 아니다.

const element = <h1>안녕하세요</h1>;

그래서 JSX를 브라우저에서 직접 실행하려면 Babel이 JSX를 JavaScript 코드로 바꿔줘야 한다.

3. JSX가 변환되는 느낌

수업에서는 아래처럼 설명하면 쉽다.

우리가 작성하는 JSX:

const element = <h1>안녕하세요</h1>;

Babel이 변환해주는 JavaScript 느낌:

const element = React.createElement(
    "h1",
    null,
    "안녕하세요"
);
정확한 변환 결과는 설정에 따라 달라질 수 있다.

하지만 초급 수업에서는 “JSX는 Babel을 거쳐 브라우저가 실행할 수 있는 JavaScript로 바뀐다”라고 이해하면 충분하다.

4. CDN 예제에서 Babel이 필요한 부분

HTML 파일에서 다음 스크립트를 추가하면 브라우저 안에서 Babel을 사용할 수 있다.

<script src="https://unpkg.com/@babel/standalone/babel.min.js"></script>

그리고 JSX가 들어있는 스크립트에는 type="text/babel"을 작성한다.

<script type="text/babel">
    function App() {
        return <h1>React 수업</h1>;
    }
</script>

이 뜻은 다음과 같다.

type="text/babel" → 이 script 안에는 JSX가 들어있다. → Babel이 먼저 변환한 뒤 실행한다.

5. 실제 프로젝트에서는 Babel을 직접 넣지 않는 경우가 많다

Vite, Next.js, Expo 같은 프로젝트 도구를 사용하면 내부적으로 JSX와 최신 JavaScript 변환을 처리해준다.

CDN 실습: 직접 Babel script 추가 Vite / Next.js / Expo 프로젝트: 개발 도구가 내부적으로 변환 처리
Babel 핵심 정리
  • Babel은 JavaScript 컴파일러이다.
  • JSX는 브라우저가 그대로 실행할 수 없으므로 변환이 필요하다.
  • CDN 실습에서는 @babel/standalone을 사용한다.
  • type="text/babel"은 해당 스크립트를 Babel로 변환하겠다는 의미이다.
  • 실제 프로젝트에서는 Vite, Expo 같은 도구가 변환 과정을 대신 처리한다.

Chapter 10
React Hook 기본 규칙

1. Hook이란?

Hook은 함수 컴포넌트에서 React 기능을 사용할 수 있게 해주는 함수이다.

Hook 이름은 보통 use로 시작한다.

useState 상태 관리 useEffect 렌더링 이후 실행할 작업 useRef 값 또는 참조 저장 useMemo 계산 결과 재사용 useCallback 함수 재사용 useContext 여러 컴포넌트가 값 공유

2. Hook 사용 규칙

  • Hook은 함수 컴포넌트의 최상위에서 호출한다.
  • 조건문 안에서 Hook을 호출하지 않는다.
  • 반복문 안에서 Hook을 호출하지 않는다.
  • 중첩 함수 안에서 Hook을 호출하지 않는다.
  • Hook 이름은 use로 시작한다.

잘못된 예:

function App() {
    if (true) {
        const [count, setCount] = useState(0);
    }

    return <h1>잘못된 예</h1>;
}

올바른 예:

function App() {
    const [count, setCount] = useState(0);

    if (count > 0) {
        console.log("count가 0보다 큽니다.");
    }

    return <h1>{count}</h1>;
}

1. useContext란?

useContext는 여러 컴포넌트에서 공통 값을 쉽게 사용하게 해준다.

예를 들어 로그인 사용자 정보, 테마 정보, 언어 설정 등을 여러 컴포넌트에서 사용할 때 쓴다.

const UserContext = React.createContext();

function App() {
    const user = {
        name: "홍길동",
        role: "admin"
    };

    return (
        <UserContext.Provider value={user}>
            <Profile />
        </UserContext.Provider>
    );
}

function Profile() {
    const user = React.useContext(UserContext);

    return (
        <div>
            <h1>{user.name}</h1>
            <p>{user.role}</p>
        </div>
    );
}

  • 여러 컴포넌트가 같은 값을 사용할 때 쓴다.
  • props를 여러 단계로 전달하지 않아도 된다.
  • 로그인 정보, 테마, 언어 설정 등에 자주 사용한다.

Chapter 17
Custom Hook

1. Custom Hook이란?

Custom Hook은 개발자가 직접 만드는 Hook이다.

이름은 반드시 use로 시작해야 한다.

useCounter useInput useUser useFetch

2. useCounter 예제

function useCounter() {
    const [count, setCount] = React.useState(0);

    function increase() {
        setCount((prev) => prev + 1);
    }

    function decrease() {
        setCount((prev) => prev - 1);
    }

    return {
        count,
        increase,
        decrease
    };
}

function Counter() {
    const { count, increase, decrease } = useCounter();

    return (
        <div>
            <h1>{count}</h1>
            <button onClick={increase}>증가</button>
            <button onClick={decrease}>감소</button>
        </div>
    );
}

  • 반복되는 로직을 분리할 수 있다.

  • 상태 관리 로직을 재사용할 수 있다.

수업 마무리 정리

  • React는 UI를 만들기 위한 JavaScript 라이브러리이다.
  • React는 Facebook, 현재 Meta에서 시작되었다.
  • React는 화면을 컴포넌트 단위로 나누어 만든다.
  • JSX는 JavaScript 안에서 HTML처럼 화면 구조를 작성하는 문법이다.
  • 브라우저는 JSX를 그대로 실행할 수 없으므로 Babel 같은 변환 도구가 필요하다.
  • Virtual DOM은 실제 DOM을 효율적으로 업데이트하기 위한 가상의 UI 표현이다.
  • SPA는 하나의 HTML을 기반으로 JavaScript가 화면을 바꾸는 방식이다.
  • MPA는 여러 HTML 페이지를 서버에서 받아오는 전통적인 방식이다.
  • React는 기존 HTML에 라이브러리처럼 추가할 수도 있다.
  • Expo를 사용하면 React Native 기반 앱 프로젝트를 쉽게 만들 수 있다.
  • Hook은 함수 컴포넌트에서 상태, 효과, 참조, 공유 값 등을 사용할 수 있게 해준다.
  • useState는 상태 관리에 사용한다.
  • useEffect는 렌더링 이후 실행할 작업에 사용한다.
  • useRef는 렌더링과 상관없는 값을 저장할 때 사용한다.
  • useMemo는 계산 결과를 기억할 때 사용한다.
  • useCallback은 함수를 기억할 때 사용한다.
  • useContext는 여러 컴포넌트가 같은 값을 공유할 때 사용한다.
  • Custom Hook은 반복되는 Hook 로직을 재사용하기 위해 직접 만드는 Hook이다.
